# AI Copy/Paste Buddy — Settings UI + Keychain Tech Spec

> Companion to `SPEC.md`. This document adds a small Settings window and migrates the OpenRouter API key from a plaintext JSON file into the macOS Keychain, in preparation for friends-and-family DMG distribution.

---

## 1. Goal

Add the minimum UI surface needed so a non-technical user can:

1. On **first launch** after installing the app, be presented with a window that asks for their OpenRouter API key.
2. On **every subsequent launch**, open the same window from the menu bar to view/edit/clear the key.

Store the key in the macOS Keychain so it is encrypted at rest, survives reinstall/migration, and is not visible as plaintext on disk.

The user must **never see the word "Keychain"**, never be sent to Keychain Access.app, and never get a Keychain-related system prompt (under normal conditions — see §10.3 for the one corner case).

---

## 2. Non-goals

- No hotkey-rebinding UI. Hotkeys remain hardcoded.
- No model-selection UI. Model remains hardcoded in `OpenRouterClient`.
- No Logfire support for end users. The Logfire write token is embedded **at build time** only when the developer sets `AICPB_LOGFIRE_TOKEN` in their shell environment. The DMG build strips the variable defensively, so a binary distributed to friends-and-family never contains a token and cannot emit Logfire traces under any user configuration. End users have no UI, no JSON field, and no environment variable that enables Logfire. See §6 and §11.
- No multi-account, multi-key, or per-provider configuration.
- No telemetry on settings actions.
- No Sparkle / auto-update / notarization (those are separate work for distribution).
- No preferences for screenshot quality, lasso color, etc.

---

## 4. Behavioral contract

### 4.1 First launch (no API key in Keychain)

1. App launches; menu bar item installs as today.
2. `Config.shared.load()` finds no API key.
3. `AppDelegate` calls `SettingsWindowController.shared.show()`.
4. Window appears centered on the main screen, frontmost, app activated.
5. User pastes their key, clicks **Save**. The key is stored in Keychain and `Config.shared.apiKey` is now non-nil.
6. Window closes automatically *only* on first run (i.e., when the window was opened because the key was missing). If the user closes the window without saving, the app keeps running — they can reopen Settings later via the menu bar. No nag dialog.

### 4.2 Subsequent launches (key present)

1. App launches; menu bar item installs.
2. `Config.shared.load()` finds an API key in Keychain.
3. No window is shown. The user opens Settings via menu bar at will.

### 4.3 Editing later

1. User clicks menu bar → **Settings…** (⌘,).
2. Window appears, secure text field pre-populated with the current key (rendered as bullets by `SecureField`).
3. User edits + clicks **Save**. Status flips to "Saved ✓" inline for ~1.5s, window stays open.
4. User clicks **Clear** to remove the key entirely. Status flips to "Key removed". The next AI paste will surface the existing "API key missing" error path (already implemented in `PasteController`).

### 4.4 Menu-bar state

Whenever `Config.shared.apiKey` transitions between nil and non-nil, `Config` posts `Notification.Name("aicpb.Config.didChange")`. `MenuBar` observes this and calls `refreshConfigState()`.

The previously disabled label `"⚠️ API key missing — edit ~/.config/ai-cpb/config.json"` becomes a **clickable** menu item `"⚠️ API key missing — Open Settings…"` whose action opens the Settings window.

## 5. Keychain wrapper — `Sources/aicpb/Keychain.swift`

New file. No external dependencies beyond `Security.framework`.

```swift
import Foundation
import Security

enum Keychain {
    /// Stable service name for all ai-cpb keychain items. Matches the bundle id.
    static let service = "com.yanyiphei.aicpb"

    /// Stable account name for the OpenRouter API key.
    static let openRouterAccount = "openrouter_api_key"

    enum KeychainError: Error {
        case unexpectedStatus(OSStatus)
    }

    /// Returns the value for (service, account), or nil if not present.
    static func readString(account: String) -> String? {
        let query: [String: Any] = [
            kSecClass as String:       kSecClassGenericPassword,
            kSecAttrService as String: service,
            kSecAttrAccount as String: account,
            kSecReturnData as String:  true,
            kSecMatchLimit as String:  kSecMatchLimitOne
        ]
        var item: CFTypeRef?
        let status = SecItemCopyMatching(query as CFDictionary, &item)
        guard status == errSecSuccess else { return nil }
        guard let data = item as? Data, let str = String(data: data, encoding: .utf8) else {
            return nil
        }
        return str
    }

    /// Inserts or updates the (service, account) entry.
    /// Throws on any non-success OSStatus.
    static func writeString(_ value: String, account: String) throws {
        let data = Data(value.utf8)
        let baseQuery: [String: Any] = [
            kSecClass as String:       kSecClassGenericPassword,
            kSecAttrService as String: service,
            kSecAttrAccount as String: account
        ]
        // Try update first.
        let updateAttrs: [String: Any] = [kSecValueData as String: data]
        let updateStatus = SecItemUpdate(baseQuery as CFDictionary, updateAttrs as CFDictionary)
        if updateStatus == errSecSuccess { return }
        if updateStatus != errSecItemNotFound {
            throw KeychainError.unexpectedStatus(updateStatus)
        }
        // Insert.
        var addQuery = baseQuery
        addQuery[kSecValueData as String]      = data
        addQuery[kSecAttrAccessible as String] = kSecAttrAccessibleAfterFirstUnlockThisDeviceOnly
        let addStatus = SecItemAdd(addQuery as CFDictionary, nil)
        if addStatus != errSecSuccess {
            throw KeychainError.unexpectedStatus(addStatus)
        }
    }

    /// Removes the (service, account) entry if present. No-op if missing.
    static func delete(account: String) throws {
        let query: [String: Any] = [
            kSecClass as String:       kSecClassGenericPassword,
            kSecAttrService as String: service,
            kSecAttrAccount as String: account
        ]
        let status = SecItemDelete(query as CFDictionary)
        if status == errSecSuccess || status == errSecItemNotFound { return }
        throw KeychainError.unexpectedStatus(status)
    }
}
```

Notes for the implementer:

- `kSecAttrAccessibleAfterFirstUnlockThisDeviceOnly` is correct: the app needs to read the key after a fresh login (e.g. it autostarts), but the key must NOT sync via iCloud Keychain.
- We do not pass `kSecUseDataProtectionKeychain` — the default keychain is correct for an unsandboxed AppKit app.
- No need to handle a "Keychain locked" state in code; on macOS the login keychain is unlocked at login. If the user manually locks it (rare power-user move), reads return nil and the app behaves as if the key is missing.

## 6. Config rewrite — `Sources/aicpb/Config.swift`

### 6.1 Logfire token source-of-truth

The Logfire write token is **no longer read from JSON**. It is supplied by the compile-time constant `LogfireBuild.token`, declared in the generated file `Sources/aicpb/LogfireToken.swift` (see §11 for how the Makefile produces it). At runtime, `Config.shared.logfire` is:

- `LogfireConfig(writeToken: t)` if `LogfireBuild.token == t` for some non-empty `t`.
- `nil` otherwise — and this is the case for every binary in any DMG distributed to end users.

`Config.logfire` becomes an immutable `let` set once in `init()`. There is no API to mutate it at runtime. `LogfireLogger` continues to read `Config.shared.logfire` exactly as today.

### 6.2 Full file

Replace the entire contents of `Sources/aicpb/Config.swift` with:

```swift
import Foundation

struct LogfireConfig {
    static let tracesEndpoint = "https://logfire-us.pydantic.dev/v1/traces"
    let writeToken: String
}

final class Config {
    static let shared = Config()

    /// Posted on the main queue after any successful mutation of `apiKey`.
    /// Observers should re-read `Config.shared.apiKey` themselves; the
    /// notification carries no userInfo.
    static let didChangeNotification = Notification.Name("aicpb.Config.didChange")

    static let configDirURL: URL = {
        let home = FileManager.default.homeDirectoryForCurrentUser
        return home.appendingPathComponent(".config/ai-cpb", isDirectory: true)
    }()

    static let configFileURL: URL = configDirURL.appendingPathComponent("config.json")

    private(set) var apiKey: String?

    /// Populated at init from the compile-time constant `LogfireBuild.token`.
    /// Always nil in DMG-distributed builds. Never user-configurable.
    let logfire: LogfireConfig?

    private init() {
        if let t = LogfireBuild.token, !t.isEmpty {
            self.logfire = LogfireConfig(writeToken: t)
        } else {
            self.logfire = nil
        }
    }

    /// Loads the API key from Keychain. If `config.json` exists and contains
    /// either a legacy `openrouter_api_key` or `logfire_write_token` field,
    /// migrates the API key into Keychain (when no Keychain entry exists)
    /// and strips both fields from disk. Logfire is no longer JSON-driven.
    func load() {
        apiKey = nil

        if let stored = Keychain.readString(account: Keychain.openRouterAccount),
           !stored.isEmpty {
            apiKey = stored
        }

        guard let data = try? Data(contentsOf: Config.configFileURL),
              var jsonObject = try? JSONSerialization.jsonObject(with: data) as? [String: Any]
        else { return }

        if apiKey == nil,
           let legacy = (jsonObject["openrouter_api_key"] as? String)?
                .trimmingCharacters(in: .whitespacesAndNewlines),
           !legacy.isEmpty {
            do {
                try Keychain.writeString(legacy, account: Keychain.openRouterAccount)
                apiKey = legacy
                NSLog("ai-cpb: migrated legacy api key from config.json into Keychain")
            } catch {
                NSLog("ai-cpb: legacy api key migration FAILED: \(error)")
            }
        }

        let hadLegacy =
            jsonObject["openrouter_api_key"] != nil ||
            jsonObject["logfire_write_token"] != nil
        jsonObject.removeValue(forKey: "openrouter_api_key")
        jsonObject.removeValue(forKey: "logfire_write_token")
        if hadLegacy {
            Self.writeJSON(jsonObject)
        }
    }

    /// Writes `key` (trimmed) into Keychain, or deletes the entry if `key`
    /// is nil or trims to empty. Updates `apiKey` and posts `didChangeNotification`.
    @discardableResult
    func setAPIKey(_ key: String?) -> Result<Void, Error> {
        let trimmed = key?.trimmingCharacters(in: .whitespacesAndNewlines)
        do {
            if let v = trimmed, !v.isEmpty {
                try Keychain.writeString(v, account: Keychain.openRouterAccount)
                apiKey = v
            } else {
                try Keychain.delete(account: Keychain.openRouterAccount)
                apiKey = nil
            }
        } catch {
            return .failure(error)
        }
        NotificationCenter.default.post(name: Config.didChangeNotification, object: nil)
        return .success(())
    }

    // MARK: - private

    private static func writeJSON(_ obj: [String: Any]) {
        do {
            try FileManager.default.createDirectory(
                at: Config.configDirURL, withIntermediateDirectories: true)
            let data = try JSONSerialization.data(
                withJSONObject: obj, options: [.prettyPrinted, .sortedKeys])
            try data.write(to: Config.configFileURL, options: .atomic)
        } catch {
            NSLog("ai-cpb: failed to rewrite config.json: \(error)")
        }
    }
}
```

Contract guarantees the implementer must preserve:

- `Config.shared.apiKey` is the **only** way other code reads the key. Do not change call sites in `PasteController`, `OpenRouterClient`, or `MenuBar`.
- `Config.shared.logfire` is the **only** way other code reads Logfire state. `LogfireLogger.swift` does not change.
- `Config.shared.load()` is idempotent and safe to call multiple times.
- After `setAPIKey(...)` returns `.success`, `Config.shared.apiKey` reflects the new value synchronously.
- `didChangeNotification` is posted from the thread that called `setAPIKey`. SwiftUI / AppKit observers must be added on the main thread (which they will be — see §8).

## 7. Settings window — `Sources/aicpb/UI/SettingsWindow.swift`

New file. Create the directory `Sources/aicpb/UI/` if it does not exist. The `Makefile` already globs `Sources/aicpb/**/*.swift` via `find Sources/aicpb -name '*.swift'`, so no Makefile change is needed for the source path.

2. An `NSWindowController` subclass called `SettingsWindowController` with a `static let shared` singleton.

### 7.1 SwiftUI view

```swift
import AppKit
import SwiftUI

private struct SettingsView: View {
    /// `true` when the window was opened because no API key existed.
    /// In that mode, a successful save closes the window automatically.
    let firstRun: Bool
    /// Closure used by the view to dismiss its hosting window.
    let onRequestClose: () -> Void

    @State private var keyText: String = Config.shared.apiKey ?? ""
    @State private var statusMessage: String? = nil
    @State private var statusIsError: Bool = false
    @State private var statusClearWorkItem: DispatchWorkItem? = nil

    var body: some View {
        VStack(alignment: .leading, spacing: 16) {
            if firstRun {
                Text("Welcome to AI-CPB")
                    .font(.title2).bold()
                Text("Paste your OpenRouter API key to get started. AI-CPB uses it to decide what to paste based on what you copied.")
                    .foregroundStyle(.secondary)
                    .fixedSize(horizontal: false, vertical: true)
            } else {
                Text("OpenRouter API key")
                    .font(.headline)
            }

            SecureField("sk-or-...", text: $keyText)
                .textFieldStyle(.roundedBorder)
                .onSubmit { save() }

            HStack(spacing: 12) {
                Link("Get a key at openrouter.ai/keys",
                     destination: URL(string: "https://openrouter.ai/keys")!)
                    .font(.callout)
                Spacer()
                if let msg = statusMessage {
                    Text(msg)
                        .font(.callout)
                        .foregroundStyle(statusIsError ? .red : .green)
                }
            }

            HStack {
                Button("Clear key", role: .destructive, action: clear)
                    .disabled(Config.shared.apiKey == nil && keyText.isEmpty)
                Spacer()
                Button("Cancel", action: onRequestClose)
                    .keyboardShortcut(.cancelAction)
                Button("Save", action: save)
                    .keyboardShortcut(.defaultAction)
                    .disabled(trimmedKey.isEmpty || trimmedKey == (Config.shared.apiKey ?? ""))
            }
        }
        .padding(20)
        .frame(width: 480)
    }

    private var trimmedKey: String {
        keyText.trimmingCharacters(in: .whitespacesAndNewlines)
    }

    private func save() {
        let result = Config.shared.setAPIKey(trimmedKey)
        switch result {
        case .success:
            if firstRun {
                onRequestClose()
                return
            }
            show(status: "Saved ✓", isError: false)
        case .failure(let err):
            show(status: "Could not save: \(err.localizedDescription)", isError: true)
        }
    }

    private func clear() {
        let result = Config.shared.setAPIKey(nil)
        switch result {
        case .success:
            keyText = ""
            show(status: "Key removed", isError: false)
        case .failure(let err):
            show(status: "Could not clear: \(err.localizedDescription)", isError: true)
        }
    }

    private func show(status: String, isError: Bool) {
        statusMessage = status
        statusIsError = isError
        statusClearWorkItem?.cancel()
        let work = DispatchWorkItem { statusMessage = nil }
        statusClearWorkItem = work
        DispatchQueue.main.asyncAfter(deadline: .now() + 1.8, execute: work)
    }
}
```

### 7.2 Window controller

```swift
final class SettingsWindowController: NSObject, NSWindowDelegate {
    static let shared = SettingsWindowController()

    private var window: NSWindow?
    private var firstRun = false

    /// Show the window. Pass `firstRun: true` only when the app is presenting
    /// this window because no API key is configured at launch — it tweaks
    /// copy and auto-closes after Save.
    func show(firstRun: Bool = false) {
        self.firstRun = firstRun
        if let w = window {
            // If already open and reused, refresh first-run behavior.
            w.makeKeyAndOrderFront(nil)
            NSApp.activate(ignoringOtherApps: true)
            return
        }
        let view = SettingsView(firstRun: firstRun, onRequestClose: { [weak self] in
            self?.window?.performClose(nil)
        })
        let hosting = NSHostingController(rootView: view)
        let win = NSWindow(contentViewController: hosting)
        win.title = "AI-CPB Settings"
        win.styleMask = [.titled, .closable, .miniaturizable]
        win.isReleasedWhenClosed = false
        win.center()
        win.delegate = self
        window = win
        win.makeKeyAndOrderFront(nil)
        NSApp.activate(ignoringOtherApps: true)
    }

    func windowWillClose(_ notification: Notification) {
        window = nil
    }
}
```

Notes:

- The window controller is intentionally a plain `NSObject`, not `NSWindowController`, to keep the implementation minimal. Both work; the spec is the source of truth.
- `isReleasedWhenClosed = false` is important — the SwiftUI hosting controller would otherwise be torn down twice.
- The window does **not** float (`win.level` defaults to `.normal`), so it can be sent behind other windows; that's fine because the app is `LSUIElement=true` and the user expects it to behave like a normal app window.
- Do not change the app's `activationPolicy`. Accessory apps can show key windows; only the Dock icon is suppressed.

func applicationDidFinishLaunching(_ notification: Notification) {
    menuBar.install()
    Config.shared.load()

## 10. Edge cases

### 10.1 Empty / whitespace-only key

`Config.setAPIKey` trims input. A whitespace-only save is treated as a delete and the Keychain entry is removed.

### 10.2 Save error

`SecItemAdd` / `SecItemUpdate` / `SecItemDelete` can return non-success `OSStatus` values. Surface a red inline message under the field via `statusMessage` — do **not** raise an `NSAlert`. The user can retry. Do not crash, do not silently swallow the error.

### 10.3 Signing-identity change → "wants to access your keychain" prompt

If the app is rebuilt with a different code-signing identity (e.g., the local `ai-cpb-local` ad-hoc cert is replaced with a real Developer ID cert later), macOS treats the new binary as a different app from the one that originally stored the Keychain item. On the first read/write after the switch, the OS will show a one-time `"ai-cpb wants to access your keychain"` dialog. This is acceptable and expected; do not attempt to suppress it. For the friends-and-family DMG, sign once with the final identity and ship that.

### 10.4 Concurrent paste during a save

`PasteController` reads `Config.shared.apiKey` once at the top of `run()`. A Save that completes mid-paste does not affect the in-flight call. No locking required.

### 10.5 Window opened twice

`SettingsWindowController.show(...)` brings the existing window forward instead of creating a second one. The `firstRun` flag is updated on each call.

### 10.6 Legacy `config.json` with API key and/or Logfire token

If a previous version wrote `openrouter_api_key` and/or `logfire_write_token` to the JSON file, `Config.load()`:

- Migrates `openrouter_api_key` (if present) to Keychain.
- Strips **both** legacy fields from the JSON, then rewrites it.
- Never reads the JSON `logfire_write_token` into runtime state — Logfire on the new build is governed entirely by the compile-time `LogfireBuild.token` constant.

If, after stripping, the file would be `{}`, it is rewritten as `{}` (empty object). The file is not deleted. The developer's Logfire setup must be reestablished via the build-time env var (§11); the JSON field is permanently dead.

### 10.7 Keychain access denied / corrupt

If `Keychain.readString` returns nil due to a denial or corruption, the app behaves as if no key is set: it opens the Settings window. The user can re-enter the key, which will overwrite/repair the entry on Save.

## 11. Build system — `Makefile` and `.gitignore`

### 11.1 `SWIFTC_FLAGS` — link Security and SwiftUI

```make
SWIFTC_FLAGS := -O \
	-target arm64-apple-macos14.0 \
	-framework AppKit \
	-framework ApplicationServices \
	-framework Carbon \
	-framework CoreGraphics \
	-framework Security \
	-framework SwiftUI
```

### 11.2 `SOURCES` — exclude the generated file from the glob

The generated `LogfireToken.swift` must not be picked up twice. Filter it out of the find result and add it back explicitly:

```make
SOURCES := $(shell find Sources/aicpb -name '*.swift' -not -name 'LogfireToken.swift')
LOGFIRE_GEN := Sources/aicpb/LogfireToken.swift
```

### 11.3 `logfire-token-gen` — regenerate every build

Add a phony target that writes `LogfireToken.swift` from one of two sources: the `AICPB_LOGFIRE_TOKEN` environment variable, or — if the env var is unset — an `AICPB_LOGFIRE_TOKEN=` line in a top-level `.env` file. Precedence: env var (even if explicitly empty) > `.env` > nil.

The "explicitly empty env var" case is load-bearing: `make dmg` invokes a recursive `$(MAKE)` with `AICPB_LOGFIRE_TOKEN=` to suppress the token. With this precedence, the suppression wins over any `.env` content, so the DMG binary cannot embed a developer token even when `.env` contains one.

The recipe must run on every build (so a change in `AICPB_LOGFIRE_TOKEN` or `.env` propagates), so declare it `.PHONY` and list it as a prerequisite of `$(EXE)`.

```make
.PHONY: build run clean install reinstall dmg logfire-token-gen

logfire-token-gen:
	@mkdir -p Sources/aicpb
	@TOKEN="$${AICPB_LOGFIRE_TOKEN-__UNSET__}"; \
	SOURCE="env var"; \
	if [ "$$TOKEN" = "__UNSET__" ]; then \
		if [ -f .env ]; then \
			TOKEN=$$(grep -E '^[[:space:]]*AICPB_LOGFIRE_TOKEN=' .env | head -1 \
				| sed -E 's/^[[:space:]]*AICPB_LOGFIRE_TOKEN=//' \
				| sed -e 's/^"//' -e 's/"$$//' -e "s/^'//" -e "s/'$$//"); \
			SOURCE=".env"; \
		else \
			TOKEN=""; \
		fi; \
	fi; \
	if [ -n "$$TOKEN" ]; then \
		printf 'enum LogfireBuild { static let token: String? = "%s" }\n' "$$TOKEN" > $(LOGFIRE_GEN); \
		echo "→ Logfire token embedded for this build (from $$SOURCE)"; \
	else \
		echo 'enum LogfireBuild { static let token: String? = nil }' > $(LOGFIRE_GEN); \
		echo "→ Logfire token absent — Config.shared.logfire will be nil"; \
	fi

$(EXE): $(SOURCES) logfire-token-gen
	@mkdir -p "$(BUILD_DIR)"
	@echo "→ Compiling Swift sources"
	@swiftc $(SWIFTC_FLAGS) $(SOURCES) $(LOGFIRE_GEN) -o "$(EXE)"
```

The `${VAR-default}` parameter expansion (single dash, not `:-`) returns `default` only when `VAR` is **unset**, not when it is set to empty. That is what gives `.env` an "only-if-unset" fallback while preserving the dmg-strip guarantee.

Logfire tokens are URL-safe (base32/base64 alphabet); no shell escaping of `"`, `\`, or newlines is required. The `.env` parser additionally tolerates optional leading whitespace and surrounding single/double quotes around the value.

### 11.4 `dmg` target — strip the env var defensively

`make dmg` must produce a binary that **cannot** contain a Logfire token even if the developer's shell exports one. Achieve this by clearing the variable in the shell prefix of a recursive `$(MAKE)`:

```make
dmg:
	@echo "→ DMG build: stripping AICPB_LOGFIRE_TOKEN for this build"
	@AICPB_LOGFIRE_TOKEN= $(MAKE) $(APP_BUNDLE)
	@echo "→ Staging DMG contents"
	@rm -rf "$(BUILD_DIR)/dmg-staging" "$(DMG)"
	@mkdir -p "$(BUILD_DIR)/dmg-staging"
	@cp -R "$(APP_BUNDLE)" "$(BUILD_DIR)/dmg-staging/"
	@ln -s /Applications "$(BUILD_DIR)/dmg-staging/Applications"
	@echo "→ Building $(DMG)"
	@hdiutil create \
		-volname "$(APP_NAME)" \
		-srcfolder "$(BUILD_DIR)/dmg-staging" \
		-ov -format UDZO \
		"$(DMG)" >/dev/null
	@rm -rf "$(BUILD_DIR)/dmg-staging"
	@echo "✓ Built $(DMG)"
```

The `AICPB_LOGFIRE_TOKEN=` prefix sets the var to empty for the recursive `$(MAKE)` invocation and all recipes it runs, including `logfire-token-gen`. The PHONY codegen step regenerates `LogfireToken.swift` with `nil`, which in turn invalidates `$(EXE)` and forces a fresh recompile. The DMG binary therefore cannot embed a token.

The previous `dmg: $(APP_BUNDLE)` direct dependency must be removed — `dmg` no longer depends on `$(APP_BUNDLE)` at the make level; it invokes the sub-make itself.

## 12. Acceptance criteria

A correct implementation must satisfy **all** of the following. An AI implementer should treat these as test cases and verify each one manually before reporting the task complete.

1. **Fresh launch** with no Keychain entry and no `config.json` → menu bar item appears AND a Settings window appears, centered, frontmost. The window has the "Welcome to AI-CPB" heading.

2. **Save flow** → pasting a non-empty string and clicking **Save** (or pressing Enter) stores it in Keychain. The window closes automatically (because it was opened in `firstRun` mode). The menu bar's "API key missing" item is now hidden.

3. **Relaunch** after step 2 → no window opens. The menu bar's `Settings…` item is reachable and opens the window. The `SecureField` is non-empty (bullets visible).

4. **Edit + Save** while window stays open → status shows "Saved ✓" for ~1.8s then disappears. Subsequent paste operations use the new key.

5. **Clear key** → Keychain entry deleted. `Config.shared.apiKey == nil`. Menu bar "API key missing — Open Settings…" item is now visible AND clicking it reopens Settings.

6. **Quit + relaunch after Clear** → Settings window opens automatically again (first-run path).

7. **Legacy migration** → with a `config.json` containing `{ "openrouter_api_key": "sk-test", "logfire_write_token": "lf-test" }` and no Keychain entry, launching the app:
   - Loads "sk-test" as `apiKey`.
   - Stores "sk-test" in Keychain.
   - Rewrites `config.json` to `{}` (empty object, pretty-printed). Both legacy fields are stripped.
   - Does NOT make `Config.shared.logfire` non-nil from JSON. (`Config.shared.logfire` is governed solely by `LogfireBuild.token`; if the build did not set the env var, it remains nil.)
   - Does not open the Settings window.

8. **Paste path unchanged** → the existing AI Copy / AI Paste hotkey flows still work end-to-end after the changes, reading the key from `Config.shared.apiKey`.

9. **No regressions in menu bar** → status flashes (copying, copied, pasting, idle) still work. `Check permissions` and `Clear copied context` items still work. The `Reveal config file` item has been removed.

10. **Build is clean** → `make clean && make build` succeeds with no warnings introduced by the new files. The resulting `.app` launches and behaves per all the above.

11. **Window behavior under accessory policy** → the Settings window appears in front when invoked from the menu bar even when other apps are frontmost. Closing the window does **not** terminate the app (menu bar stays). No Dock icon appears at any point.

12. **Logfire OFF by default** → with `AICPB_LOGFIRE_TOKEN` unset in the environment, `make clean && make build` produces a binary in which `Config.shared.logfire == nil`. `LogfireLogger` performs zero network requests during normal use. The generated `Sources/aicpb/LogfireToken.swift` literally contains `static let token: String? = nil`.

13. **Logfire ON for the developer** → with `AICPB_LOGFIRE_TOKEN=lf-xxx` exported, `make clean && make build` produces a binary in which `Config.shared.logfire?.writeToken == "lf-xxx"`. The generated file contains the token. An AI paste cycle emits a Logfire POST.

14. **DMG never embeds the token** → with `AICPB_LOGFIRE_TOKEN=lf-xxx` exported, `make dmg` produces `build/ai-cpb.dmg` whose embedded `.app` binary contains no occurrence of `lf-xxx`. The verification command in §11.6 (`strings ... | grep ...`) must print `OK`.

15. **`.gitignore` covers the generated file** → after any `make build`, `git status` does NOT show `Sources/aicpb/LogfireToken.swift` as modified or untracked.

### 13.1 Developer workflow note (for Yifei)

Two ways to supply the token, in precedence order:

1. **`.env` file in the repo root (preferred for ongoing dev)** — gitignored. Paste your token into the scaffolded line:

   ```
   AICPB_LOGFIRE_TOKEN=lf-...
   ```

   Then `make build` picks it up automatically. No shell-config changes needed.

2. **`AICPB_LOGFIRE_TOKEN` env var** — overrides `.env` for a single invocation or a whole shell:

   ```bash
   AICPB_LOGFIRE_TOKEN=lf-... make build      # one-off
   export AICPB_LOGFIRE_TOKEN="lf-..."        # persistent for the shell
   ```

Either way, `make dmg` strips the variable defensively and ignores `.env` (via the explicit-empty env var passed through recursive `$(MAKE)`), so you can `make dmg` from the same checkout without leaking the token. If you ever revoke or rotate the token, just update `.env` (or your export) and rebuild — the codegen step regenerates `LogfireToken.swift` on every build.
